关于图片使用：
1.安装url-loader，图片属于文件
2.配置webpack.config.js
2.1 大于limit的话要安装file-loader,并且由于图片会自动被编译放到dist文件夹下，使用的是dist下的图片，
这个时候，图片路径为（"dist/xxx.jpg"）,所以，需要再配置output：publicPath:"dist/",
对于任何在别的地方引用的文件，都会在前面加上publicPath的值
2.2 小于limit的话它会自动编译成base64并作为图片的路径,这个无需配置
3.如果要修改dist下的图片文件名称，则在limit下添加
name: 'img/[name].[hash:8].[ext]'
img:创建dist下的img文件夹
[name]:保留图片原名称（本来会编译成32为hash值的名称）
[hash:8]:取32位的hash值中的前8位
[ext]:图片扩展名
4.ES6语法处理
A:npm install --save-dev babel-loader@7 babel-core babel-preset-es2015
B:webpack.config.js 的配置的preset要改成 [es2015]
